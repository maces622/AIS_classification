# AIS 轨迹半监督分类深度学习框架

## 1. 项目简介

本项目是一个专门用于处理和分类船舶自动识别系统 (AIS) 数据的深度学习框架。该框架采用半监督学习模式，结合了变分自编码器 (VAE) 和卷积神经网络 (CNN)，能够在有标签数据有限的情况下，利用大量无标签数据对船舶的轨迹进行特征提取，并将其分类为不同的船舶类型。

## 2. 数据来源与范围

本框架默认使用的测试数据来自 NOAA (美国国家海洋和大气管理局)。

* **时间范围**: 2021年和2022年的1月至3月。
* **空间范围**: 美国加利福尼亚海岸区域，经度（X）范围为 -121.57 至 -117.00，纬度（Y）范围为 31.5 至 35.0。

## 3. 环境依赖

运行本框架需要以下核心 Python 库：

* `numpy`
* `pandas`
* `scipy` (用于插值计算)
* `tensorflow` / `keras` (模型构建与训练)
* `matplotlib` (用于数据和结果的可视化)

## 4. 文件结构与核心模块

### 4.1 数据预处理 1: `prco_origin_csv.py`

该脚本用于处理原始的 AIS CSV 文件流。

* **功能**: 读取 `./CA_DATA` 文件夹下的所有 CSV 文件，提取目标字段（MMSI、时间戳、经纬度、速度SOG、航向COG、船长、船宽、吃水、船舶类型）。
* **输出**: 按照 MMSI 和时间戳进行升序排序后，将数据保存为 `california_1.pkl` 文件。

### 4.2 数据清洗与重采样 2: `filt_data.py`

该脚本旨在按照 CNN 模型的输入要求深度清理和格式化 AIS 数据。

* **轨迹切分**: 如果相邻数据点时间差大于2小时或跨越了日期，则将轨迹拆分为两段。
* **异常过滤**: 剔除最大速度不足 1.0、低速点占比过高、轨迹总时间小于6小时或数据点少于160个的异常轨迹。
* **类别映射**: 剔除未知类型的船舶，将合格船舶映射为 4 个主要类别短码：渔船（代号3）、客船（代号6）、货船（代号7）和油轮（代号8）。
* **插值重采样**: 将轨迹统一重采样至目标长度 (160)；动态参数（经纬度、速度等）采用线性插值，静态参数（长宽等）采用最近邻分配。
* **输出**: 生成包含重采样合格数据的 `procd_california.pkl` 和记录原始地理轨迹的 `org_california.pkl`。

### 4.3 半监督模型训练 3: `model.py`

本项目的核心深度学习网络，实现了端到端的特征学习与分类。

* **数据编码化**: 通过 `convert_to_one_hot` 函数，将连续的 AIS 特征转换为类似图像的独热码 (One-hot) 张量输入，输入张量维度设定为 `(160, 400, 1)`。
* **网络架构**:
* **Classifier (分类器)**: 通过堆叠的二维卷积层 (`Conv2D`) 和带有 L1/L2 正则化的全连接层提取特征，输出4个船舶类别的 Softmax 预测。
* **Encoder (编码器)**: 融合输入轨迹和标签信息，通过重参数化技巧生成维度为 6 的潜在空间变量 (Latent Variable `z`)。
* **Decoder (解码器)**: 利用潜在变量和标签，通过一系列反卷积层 (`Conv2DTranspose`) 重构原始的输入轨迹数据。


* **损失函数设计**: 半监督学习机制的核心。有标签数据的损失由二元交叉熵重构损失和 KL 散度组成；无标签数据的损失则通过穷举所有可能类别的期望损失和预测熵来计算。整体再联合分类器的交叉熵损失进行反向传播优化。
* **输出**: 打印混淆矩阵、绘制 3D 潜在空间聚类图，并将目标船舶的 MMSI 与预测分类结果持久化输出至 `mmsi_vessel_type.csv` 文件中。

## 5. 执行流程

1. 将原始 AIS 数据的 CSV 文件放置在项目根目录的 `./CA_DATA` 文件夹中。
2. 运行预处理脚本：`python prco_origin_csv.py`。
3. 运行过滤及重采样脚本：`python filt_data.py`。
4. 运行模型开始训练和预测评估：`python model.py`。
